Kidsights Data Platform
Five-minute orientation

A multi-source ETL platform combining a Nebraska childhood-development survey (REDCap) with national datasets (ACS, NHIS, NSCH) to support post-stratification weighting, multiple imputation, and IRT psychometric scoring.

01 · The eight pipelines

Each runs independently. All write to the same DuckDB.

No automatic chaining between pipelines — they were designed to run on different cadences. NE25 frequently as new survey data arrives; ACS/NHIS/NSCH annually; raking and imputation on demand.

NE25 Production

Nebraska 2025 REDCap childhood-development survey, 4 projects.

run_ne25_pipeline.R

Minnesota 2026 REDCap, multi-child households via NORC. Raking and imputation deferred.

02 · System architecture

Hybrid R + Python design, with Apache Arrow Feather as the bridge.

R handles statistical work; Python owns the database. Introduced September 2025 to eliminate persistent R DuckDB segmentation faults. The two sides exchange data via Apache Arrow Feather files.

04 · Active in-flight work

NE25 MIBB Variance Estimation — Complete

Shipped April 2026

Multiple Imputation + Bayesian Bootstrap for the sampling variance of NE25 raking weights. All three buckets of the MIBB framework have shipped; the database now has a full per-(m, b) replicate weight set for Rubin's-rules variance pooling.

  • Scale: 5 imputations × 200 Bayesian-bootstrap draws = 1,000 Stan refits, all converged (stan_ok = TRUE). Wall-clock ~3 h on 8 future::multisession workers with Stan pre-compiled in the parent session.
  • Framework: MI + sample-only Bayesian bootstrap (NE22 pattern). Targets treated as fixed population quantities; bbw ~ Exp(1) enters the masked factorized moment loss as a multiplicative data weight, not as a prior concentration — avoiding the Dirichlet-prior gradient singularity of the earlier attempt.
  • Storage: long-format DuckDB table ne25_raked_weights_boot with PK (pid, record_id, imputation_m, boot_b) — 2,645,000 rows. Python-based loader (36_store_bootstrap_weights_long.py) bulk-inserts via PyArrow after an earlier R dbAppendTable path segfaulted on the 2.6M-row write.
  • Stability: per-imputation Kish-N CV across bootstrap draws = 0.009-0.010; baseline reproducibility (bbw = 1) within ~3e-2 RMS of Bucket 2.
  • Next up (separate ticket, not started): wire reports/ne25/helpers/model_fitting.R to iterate over (imputation_m, boot_b) pairs and pool variance via Rubin's rules. Today it consumes only the single point-estimate calibrated_weight column.
  • Documentation: RAKING_INTEGRATION.md Bayesian-Bootstrap section, WEIGHT_CONSTRUCTION.qmd §5.4, todo/ne25_weights_roadmap.md (all buckets marked complete).

06 · Critical DB drift

Six divergences worth investigating before trusting any doc.

Each is a real difference between what the docs claim and what the live database contains. Suspected causes are noted, but each needs maintainer judgment about whether the DB is wrong (regenerate) or the docs are wrong (update).

1
raking_targets_ne25 has 11 rows in DB but docs claim 180 (30 estimands × 6 ages). Possibly rebuilt or dropped.
2
calibration_dataset_2020_2025 is 9,319 records in DB but docs claimed 47,084. Likely a QA-filtered subset took over.
3
ne25_calibration is documented as Step 11 output in the NE25 pipeline but does not exist in the current DB. Step may have been disabled or renamed.
4
Per-study ne**_calibration tables (NE20, NE22, NE25, NSCH21, NSCH22, USA24) do not exist. Possibly consolidated into the combined calibration_dataset_* family.
5
overall_influence_cutoff and authenticity_weight columns are documented in ne25_transformed but missing from the current schema. Likely removed in a refactor.
6
NSCH table-naming inconsistency: years 2017-2020 and 2023 use nsch_{year}_raw; years 2021 and 2022 use nsch_{year} (no _raw suffix). Both load correctly but worth standardizing.

08 · Setup quick-glance

Five steps to a working local install.

Clone the repo. Standard git clone https://github.com/marcus-waldman/Kidsights-Data-Platform.
Copy .env.template → .env and fill in IPUMS_API_KEY_PATH, REDCAP_API_CREDENTIALS_PATH, FRED_API_KEY_PATH, PYTHON_EXECUTABLE.
Get API keys: IPUMS at account.ipums.org/api_keys; REDCap from the UNMC admin; FRED at fredaccount.stlouisfed.org/apikeys.
Run python scripts/setup/verify_installation.py to confirm all checks pass.
Smoke test: R --slave --no-restore --file=run_ne25_pipeline.R. If it completes, you're operational.
Full walkthrough → docs/setup/INSTALLATION_GUIDE.md
